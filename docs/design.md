# local-UNO デザインシステム（UI 正典）

ゲーム盤 UI のデザイン正典。**2人対戦・LAN 内専用**の UNO を、**PC とスマートフォン双方の操作性**を軸に設計する。以降の web UI 作業（[#89](https://github.com/odasatoshi/localUNO/issues/89) のリデザイン実装ほか）は本書を参照する。ゲームのルールは [`house-rules.md`](./house-rules.md)、設計原則は [`spec.md`](./spec.md)。

- 検討モックアップ（承認済み・視覚確認用）: https://claude.ai/code/artifact/0e750f2a-4cb5-4da2-890e-84b8107f68e8
- トーン: **Playful**（カラフル・楽しい）。カードの4色を主役にし、UI クロームは静かに保つ。

## 0. 目的と対象

- **対象**: 同一 LAN 上の2人。多くの場合、片方が PC・片方がスマホ。
- **画面の仕事**: 1手番を素早く操作する（見る → 手札を選ぶ → 引く/出す/パス → 必要なら UNO 宣言/指摘）。読み物ではなく**操作する盤面**。
- **最優先**: PC とスマホ双方で崩れず、片手（スマホ）でも快適に操作できること。

## 1. 操作性の原則（最優先）

- **親指ゾーンに主操作**（スマホ）: 「引く / 出す / パス」を画面下の固定アクションバーに置く。片手・親指だけで手番を完結できる。
- **UNO 系は分離**: 「UNO! / UNO言ってない!」は主操作から離した段に置き、誤爆を防ぐ。
- **大きなタップ標的**: 主ボタン 52px、UNO 系 44px 以上。手札カードは重ね扇でも十分な幅を確保。
- **手番がひと目で**: 自分の番はステータスをアクセント色＋点滅ドットで強調。相手の番は主操作ボタンを無効化して誤操作を防ぐ。
- **選択の可視化**: 選んだ札は持ち上がり、送信順バッジ（1・2…）を表示。「出す（N枚）」に枚数を反映（複数枚出し）。
- **PC はホバー＋集約**: 手札はホバーで浮上、操作は下部の横1段ツールバーに集約して視線移動を最小化。キーボードフォーカスリング必須。

## 2. カラートークン（OKLCH）

中立色はアクセント（グレープ紫）へわずかに寄せる＝「選ばれた中立色」。カードの意味色（赤/黄/緑/青）はアクセントとは別軸で、UI クロームと衝突させない。

## 3. タイポグラフィ

日本語表示のため webフォント（外部 CDN）は使わず、**システム丸ゴシック系スタック**を採用（Playful な丸みを OS のフォントで得る）。

- 表示/UI: `--font-ui: "SF Pro Rounded", ui-rounded, "Hiragino Maru Gothic ProN", "Zen Maru Gothic", "Segoe UI", system-ui, sans-serif`
- 本文/データ: `--font-body: system-ui, "Hiragino Kaku Gothic ProN", "Yu Gothic", Meiryo, sans-serif`
- 見出し・表示は**必ずローマン**（`font-style: normal`）。強調は太さ・アクセント色で。
- 数字（枚数・山札残・カード数字）は `font-variant-numeric: tabular-nums`。

## 4. スペーシング・角丸・モーション・影

- スペーシング（4pt）: `--s1:4 --s2:8 --s3:12 --s4:16 --s5:24 --s6:32 --s7:48`（px）。要素間隔は flex/grid の `gap` で作る（個別 margin の相殺を避ける）。
- 角丸: `--radius: 16px`（面）、カード 11px、ボタン 14px、ピル 999px。
- イージング: `--ease-out: cubic-bezier(.22,.61,.36,1)` / `--ease-in-out: cubic-bezier(.65,.05,.36,1)`。ブラウザ既定 `ease` やバウンスは使わない。
- モーションは `transform`/`opacity` のみ・控えめに。手番ドットの点滅、選択カードの浮上、シートの pop-in 程度。`prefers-reduced-motion: reduce` で実質停止。
- 影: `--shadow-1`（面・カード）/ `--shadow-2`（持ち上がり・オーバーレイ）。ダークは不透明度を上げる。

## 5. レイアウト

**3ゾーン構成**（上=相手 / 中=場 / 下=自分＋操作）を、**同一 DOM ＋コンテナクエリ**で PC/スマホに出し分ける（マークアップを二重管理しない）。

- 盤面ラッパに `container: board / inline-size` を付け、`@container board (min-width: 720px)` を PC レイアウトの分岐点とする。
- **スマホ（既定 / < 720px）**: 縦積み。相手は上部の枚数＋裏面ミニ札。場は捨て山・山札・強制色を中央。手札は横スクロールの扇。操作は**下部固定バー**（引く/出す/パスを大きく、UNO 系はその上段）。
- **PC（≥ 720px）**: 横広。相手を上部に、場を中央大きく、手札を下部に扇状展開。操作は**下部横1段ツールバー**に集約（UNO 系＝左、主操作＝右寄せ、リセット＝ヘッダ）。

**UNO カードの造形**: 角丸矩形（`aspect-ratio: 63/88`）＋白い斜めオーバル（`border-radius: 50%/42%; transform: rotate(-22deg)`）＋中央の大きな記号＋角インデックス。CSS のみで描画し画像アセット不要。ワイルドは4色コニックグラデーション、裏面は暗色＋「UNO」ロゴ。記号の大きさは `container-type: inline-size` ＋ `cqw` でカード幅に追従。

## 6. コンポーネントと全状態

- **ボタン**: `.btn-primary`（グレープ・主操作）/ `.btn-secondary`（面・引く/パス）/ `.btn-uno`（赤・UNO!）/ `.btn-uno.ghost`（UNO言ってない!）。全要素で default / hover / `:focus-visible` / `:active` / disabled を用意。無効時は `opacity:.4` かつ hover 無効。
- **手番ステータス**: `data-turn="you"` でグレープ背景＋点滅ドット、それ以外は静かなピル。
- **手札選択**: タップ/クリックで選択トグル → 浮上＋リング＋順序バッジ。サーバから新 state を受けたら選択はクリア（stale 防止・サーバ権威）。
- **色えらび（ワイルド）**: 盤面上のオーバーレイシートに4色の大きなボタン（56px）。
- **バナー**: 勝ち（`あなたの勝ち！`）/ 負け / **引き分け（`山切れ — 引き分け`, `is_draw`）**。オーバーレイで pop-in、終局時は UNO 系ボタンを隠す。
- **UI が表出すべきサーバ状態**（[`spec.md` §5 PlayerView](./spec.md)）: current_player / awaiting / 相手手札枚数 / 山札残 / 捨て山トップ / **forced_color（強制色）** / **pending_draw（累積ドロー）** / **uno_declared** / **is_draw** / **drawn_card_id（ドロー後フェーズ→パス活性）**。

## 7. アクセシビリティ

- タップ標的: 主 52px・UNO 系 44px 以上。
- フォーカス: `:focus-visible` に `outline: 3px var(--focus)`。リングの出現はアニメーションさせない（即時表示）。
- `prefers-reduced-motion: reduce` で空間移動を停止。
- カード4色はライト/ダーク双方で判別可能なコントラストを維持。黄カードは濃色文字（`--on-yellow`）。
- 横スクロールは手札のみ。ページ本体は `overflow-x: clip`（`html`/`body`）。

## 8. 実装制約（重要・非交渉）

見た目・レイアウト/インタラクション層のみ差し替える。**機能結線とサーバ権威の描画契約は保持**する。

- **要素 ID を保持**（`app.js` と `tests/test_web.py` が依存）: `me · opponent-count · discard-top · draw-count · forced-color · hand · play-btn · draw-btn · pass-btn · reset-btn · uno-btn · challenge-btn · color-picker · banner · status`。
- **アクション契約を保持**: クライアントは Action の JSON を送るだけ（`play`/`draw`/`pass`/`choose_color`/`declare_uno`/`challenge_uno`/`reset`）。可否・効果・手番の判定はサーバ（原則1）。フロントはロジックを持たない。
- **サーバ権威の描画**: 受信した PlayerView を丸ごと描き直す。差分描画はしない。
- **テスト維持**: `tests/test_web.py` の結線検証（構造検証＋WS往復）はグリーンを保つ。UI 変更で ID/結線が変わる場合はテストも同時に更新する。
- 依存を増やさない（バニラ HTML/CSS/JS、ビルド無し, [`spec.md` §9](./spec.md)）。webフォント CDN は使わない。

## 9. トークン一覧（実装時のコピー元）

実装では `:root` に以下を宣言し、コンポーネントは必ずトークン名で参照する（生値のインライン禁止）。カラーは §2、スペーシング/角丸/モーション/影は §4 を参照。承認済みモックアップ（Artifact）が全トークンの参照実装。
